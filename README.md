# 📒التسوية البنكية - أداة التسوية البنكية

أداة ويب بسيطة واحترافية لعمل **التسوية البنكية (Bank Reconciliation)**: ترفع كشف الحساب البنكي والكشف الدفتري (الداخلي)، وتقوم الأداة تلقائيًا بـ:

- مطابقة الحركات المتشابهة في المبلغ (وبالتاريخ إن رغبت) وشطبها كـ **متطابقة** ✓
- إظهار قائمة بالحركات **الموجودة في الكشف البنكي فقط**
- إظهار قائمة بالحركات **الموجودة في الكشف الدفتري فقط**
- إتاحة **مطابقة يدوية** للحالات التي لا يلتقطها المحرك تلقائيًا
- تصدير **تقرير Excel** كامل بنتائج التسوية

كل المعالجة تتم **محليًا داخل المتصفح** — لا يتم رفع أي ملف أو بيانات إلى أي خادم، وهي مناسبة لبيانات مالية حساسة.

---

## 🚀 تجربة الأداة مباشرة

1. افتح ملف `index.html` مباشرة في المتصفح (بالنقر المزدوج عليه)، أو
2. فعّل **GitHub Pages** بعد رفع المشروع (راجع القسم أدناه) للحصول على رابط مباشر تشاركه مع فريقك.

للتجربة السريعة، استخدم الملفين الموجودين في مجلد `sample-data/`:
- `كشف_بنكي_تجريبي.csv`
- `كشف_دفتري_تجريبي.csv`

---

## 🧭 طريقة الاستخدام

1. **ارفع الملفين**: الكشف البنكي والكشف الدفتري (بصيغة `.xlsx` أو `.xls` أو `.csv` أو `.pdf`).
2. **حدّد الأعمدة**: تحاول الأداة تخمين أعمدة (التاريخ، البيان، المبلغ) تلقائيًا — تحقق منها أو عدّلها. يمكنك تحديد عمود "مبلغ" واحد بقيمة موجبة/سالبة، أو عمودي "مدين/دائن" منفصلين.
3. **اضبط الخيارات (اختياري)**:
   - **نطاق سماحية التاريخ**: عدد الأيام المسموح بها كفارق بين تاريخ الحركة في الكشفين حتى تُعتبر مطابقة (مفيد عند تأخر قيد الحركة بيوم أو يومين).
   - **سماحية فرق المبلغ**: فرق بسيط مسموح به بين المبلغين (مثلاً لفروقات تقريب).
4. اضغط **تنفيذ المطابقة**.
5. راجع النتائج عبر التبويبات الثلاثة: **المتطابقة**، **بالبنكي فقط**، **بالدفتري فقط**.
6. لأي حركتين تعرفهما كمطابقتين رغم عدم التقاطهما تلقائيًا: اضغط على الحركة في تبويب "بالبنكي فقط" وأخرى في "بالدفتري فقط"، ثم اضغط **طابق يدويًا**.
7. اضغط **تصدير تقرير التسوية** للحصول على ملف Excel كامل بالنتائج.

---

## ⚙️ كيف تعمل المطابقة تقنيًا

- تُحوَّل كل حركة إلى قيمة مطلقة مقربة لخانتين عشريتين.
- يُقارن كل سجل من الكشف البنكي بالسجلات غير المطابقَقة بعد في الكشف الدفتري، ويُختار أقرب سجل من حيث **المبلغ** (ثم **التاريخ** إن وُجد أكثر من احتمال).
- عند العثور على مطابقة، تُستبعد الحركتان من قائمة "غير المطابق" وتُدرجان في تبويب **المتطابقة**.
- الحركات المتبقية في كل كشف تظهر في تبويبها الخاص كملاحظة تستدعي المراجعة اليدوية.

> ⚠️ هذه أداة مساعدة لتسريع عملية التسوية وتقليل المراجعة اليدوية — وليست بديلاً عن المراجعة المحاسبية النهائية من قِبل شخص مختص.

## 📄 دعم ملفات PDF

تستطيع الأداة قراءة كشوفات بصيغة PDF مباشرة (عبر مكتبة [pdf.js](https://mozilla.github.io/pdf.js/)) دون الحاجة لتحويلها يدويًا:

- تُستخرج الحركات من كل سطر يبدأ بتاريخ ويحتوي على مبلغ.
- أول رقم مالي بعد التاريخ في كل سطر يُعتبر **مبلغ الحركة**؛ أي أرقام أخرى بعده (مثل الرصيد المتراكم) تُتجاهل.
- **مدعوم**: ملفات PDF نصّية (المُصدَّرة مباشرة من نظام البنك الإلكتروني — النص فيها قابل للتحديد والنسخ).
- **غير مدعوم حاليًا**: ملفات PDF عبارة عن صور ممسوحة ضوئيًا (بلا طبقة نص). في هذه الحالة ستظهر رسالة توضح ذلك — استخدم أداة OCR لتحويلها إلى نص أولًا، أو اطلب من البنك نسخة Excel/CSV.
- نظرًا لاختلاف تنسيقات كشوفات البنوك، راجع دائمًا الحركات المستخرجة في خطوة "تحديد الأعمدة" قبل تنفيذ المطابقة، خصوصًا حقل "البيان" الذي قد يحتاج تدقيقًا بسيطًا.

## 🗂️ هيكل المشروع

```
bank-reconciliation/
├── index.html          # الصفحة الرئيسية
├── css/
│   └── style.css       # التنسيق (هوية بصرية بطابع دفتر الأستاذ)
├── js/
│   └── app.js           # منطق القراءة، المطابقة، والتصدير
├── sample-data/          # ملفات تجريبية للاختبار
└── README.md
```

لا حاجة لأي خطوة بناء (build step) أو تثبيت حزم — الأداة تعمل بملفات HTML/CSS/JS خام، وتستخدم مكتبة [SheetJS](https://sheetjs.com/) عبر CDN لقراءة وكتابة ملفات Excel.

## 📤 رفع المشروع على GitHub

```bash
cd bank-reconciliation
git init
git add .
git commit -m "الإصدار الأول: أداة التسوية البنكية"
git branch -M main
git remote add origin https://github.com/USERNAME/REPO_NAME.git
git push -u origin main
```

استبدل `USERNAME/REPO_NAME` باسم المستخدم واسم المستودع الخاص بك على GitHub.

### تفعيل رابط مباشر (GitHub Pages)

1. من المستودع على GitHub، اذهب إلى **Settings → Pages**.
2. تحت **Branch** اختر `main` والمجلد `/ (root)`، ثم احفظ.
3. بعد دقيقة أو دقيقتين، سيصبح رابط الأداة متاحًا على:
   `https://USERNAME.github.io/REPO_NAME/`

## 🛠️ التطوير لاحقًا

المشروع بسيط عن قصد ليسهل التعديل عليه تدريجيًا:

- منطق المطابقة بالكامل في `js/app.js` داخل الدالة `runMatching()`.
- التنسيق والألوان محددة كمتغيرات CSS في أعلى `css/style.css` (`:root`) لتسهيل تغيير الهوية البصرية.
- لإضافة صيغ ملفات أخرى أو حقول إضافية (مثل رقم المرجع/الشيك) عدّل قسم "تحديد الأعمدة" (`ROLE_OPTIONS` و `buildRecords`).

أفكار للتوسعة المستقبلية:
- مطابقة بالاعتماد على رقم المرجع/الشيك بالإضافة إلى المبلغ.
- حفظ نتائج التسوية بين الجلسات.
- دعم رفع أكثر من ملف لكل جهة (تجميع عدة أشهر).
